在Git中，标签是非常有用的特性，它允许你将某个特定的提交点标记为重要的里程碑，如版本发布。使用标签，你可以快速地引用版本历史中的特定点，而不需要记住复杂的提交哈希值。

标签在Git中通常用于标记发布版本（如v1.0, v2.0等）。有两种类型的标签：

- **轻量标签（Lightweight）**：就像一个不会改变的分支，它只是指向特定提交的引用。
- **注释标签（Annotated）**：存储在Git数据库中的完整对象，它们包含标签创建者的名字、电子邮件、日期，并且可以有一条附加的标签信息；可以被GPG签名和验证。

## 1、查看标签

要查看仓库中的所有标签，可以使用`git tag`命令。这个命令会列出所有标签的列表：

```bash
git tag
```

### （1）查看特定标签

如果你的仓库中有很多标签，你可以使用`git tag`命令加上搜索模式来查找特定的标签。例如，要查找所有的v1.4系列标签，可以使用如下命令：

```bash
git tag -l "v1.4*"
```

这会列出所有以"v1.4"开头的标签。

### （2）查看标签详情

对于注释标签，你可以使用`git show`命令来查看标签的详细信息，包括标签的创建者信息，标签消息，以及该标签所指向的提交：

```bash
git show <tagname>
```

将`<tagname>`替换为你想要查看的标签名。

通过查看标签，你可以很容易地确定特定的发布版本或重要的开发里程碑。这在管理大型项目或跟踪已发布版本时尤其有用。

## 2、创建标签

创建标签是Git中标记重要历史点的一个重要功能，尤其是用于版本发布。Git支持两种类型的标签：轻量标签和注释标签。

### （1）创建轻量标签

轻量标签本质上是一个指向特定提交的引用，它像一个不可变的分支。创建轻量标签非常简单：

```bash
git tag <tagname>
```

这里的`<tagname>`是你选择的标签名称。例如，创建一个轻量标签`v1.0`指向当前提交，只需执行：

```bash
git tag v1.0
```

轻量标签通常用于私人用途，标记临时状态，因为它不包含其他信息。

### （2）创建注释标签

注释标签是存储在Git数据库中的一个完整对象。它包含了标签信息，标签的创建者信息（名字和邮件），创建日期，有一条附加消息，而且它可以被签名和验证。创建注释标签，使用：

```bash
git tag -a <tagname> -m "<tagmessage>"
```

- `-a <tagname>`：指定标签名
- `-m "<tagmessage>"`：附加的消息，描述标签的用途或其他重要信息

例如，创建一个注释标签`v2.0`：

```bash
git tag -a v2.0 -m "Release version 2.0"
```

### （3）指定提交创建标签

默认情况下，标签会被附加到当前HEAD所指向的提交上。但是，你可以为标签指定任何一个提交。只需在创建标签的命令中加上提交的哈希值：

```bash
git tag -a v1.0.1 <commit-hash> -m "Patch release 1.0.1"
```

这里的`<commit-hash>`是你想要标记的那个提交的哈希值。

### （4）使用GPG签名标签

如果你已经配置了GPG签名，可以创建一个经过GPG签名的标签，以证明该标签的创建者确实是你。使用`-s`选项代替`-a`：

```bash
git tag -s <tagname> -m "<tagmessage>"
```

这会要求你输入GPG密码来签名标签。

## 3、推送标签

在Git中，创建标签后，默认情况下标签不会自动同步到远程仓库。推送标签到远程仓库是一个显式的操作，这允许团队成员共享标签，用于标记发布版本或其他重要事件。

### （1）推送单个标签

要推送一个特定的标签到远程仓库，使用`git push`命令，指定远程仓库的名称（通常是`origin`）和标签的名称：

```bash
git push origin <tagname>
```

这里的`<tagname>`是你想推送的标签的名称。例如，要推送名为`v1.0`的标签，命令将是：

```bash
git push origin v1.0
```

### （2）推送所有本地标签

如果你想一次性推送本地仓库中的所有标签到远程仓库，可以使用：

```bash
git push origin --tags
```

这个命令会将你所有的本地标签推送到远程仓库。这很方便，但需要注意不要推送不再需要或过时的标签。

### （3）注意事项

- 在推送标签之前，确保标签的名称和指向的提交是你想要的。一旦标签被推送到远程仓库，它可能会被其他团队成员拉取或用于构建。
- 如果你的标签是基于重要的里程碑，如产品发布，确保在推送前与团队沟通，避免任何混淆或错误。
- 使用`--tags`选项推送所有标签很方便，但请确保不要推送任何未准备好的或临时的标签。

## 4、检出标签

在Git中，检出标签允许你查看仓库在特定标签所指向的时间点的状态。这在回溯历史版本、审查代码或验证旧版本的行为时非常有用。由于标签是对提交的引用，检出标签实质上意味着将HEAD指向那个提交，但请注意，这会让你的工作目录处于“分离HEAD状态”（detached HEAD state）。

### （1）检出标签到分离的HEAD

要检出特定的标签，使用`git checkout`命令后跟标签名：

```bash
git checkout <tagname>
```

例如，要检出名为`v1.0`的标签：

```bash
git checkout v1.0
```

这会将你的HEAD指向`v1.0`标签所指的提交。此时，你处于分离HEAD状态，意味着任何新的提交都不会属于任何分支，如果切换回分支，这些提交可能会丢失。

### （2）在新分支上检出标签

如果你想基于一个标签进行开发或修复，最佳实践是在检出标签时创建一个新分支。这样可以避免分离HEAD状态的问题。使用以下命令可以做到这一点：

```bash
git checkout -b <branchname> <tagname>
```

这里，`<branchname>`是你想要创建的新分支的名称，`<tagname>`是标签的名称。例如，如果你想基于`v1.0`标签创建一个名为`fixes-for-v1.0`的分支：

```bash
git checkout -b fixes-for-v1.0 v1.0
```

这个命令会创建一个新分支，并且自动切换到这个新分支上。

### （3）使用`git switch`检出标签（Git 2.23及以上版本）

从Git 2.23开始，`git checkout`命令被分为两个更专门的命令：`git switch`和`git restore`。要检出标签到一个新分支，可以使用`git switch`命令：

```bash
git switch -c <branchname> <tagname>
```

这与使用`git checkout`创建新分支的效果相同，但语法更清晰。

### （4）注意事项

- 检出标签到分离HEAD状态对于临时审查代码很有用，但应避免在这种状态下进行开发。
- 创建新分支来继续基于标签的开发是一种安全的做法，可以确保你的更改被正确记录在版本历史中。
- 在分离HEAD状态下做出的任何提交如果没有被新分支引用，可能会成为垃圾收集的对象并最终被清除。

## 5、删除标签

在Git中，有时候你可能需要删除一个标签。这可能是因为标签被错误地创建或命名，或者因为某个版本不再被需要。无论是需要删除本地标签还是远程标签，Git都提供了相应的命令来完成这项任务。

### （1）删除本地标签

要删除本地仓库中的一个标签，可以使用`git tag`命令加上`-d`选项。例如，如果你想删除名为`v1.0`的标签，你可以使用以下命令：

```bash
git tag -d v1.0
```

这个命令会从你的本地仓库中删除指定的标签。你将看到一个消息，确认标签已被删除。

### （2）删除远程标签

如果你已经推送了一个标签到远程仓库，并希望删除它，首先你需要删除本地标签，然后使用`git push`命令加上`--delete`选项来删除远程标签。假设你想删除远程仓库中的`v1.0`标签，你需要执行以下两个命令：

1. 删除本地标签（如果之前没有删除）：

    ```bash
    git tag -d v1.0
    ```

2. 删除远程标签：

    ```bash
    git push --delete origin v1.0
    ```

这里，`origin`是远程仓库的名称（`origin`是默认名称），`v1.0`是要删除的标签名称。执行这个命令后，远程标签将被删除。

### （3）注意事项

- 在删除标签之前，请确保这个标签确实不再需要。删除标签可能会影响依赖于这个标签的构建或部署过程。
- 如果你的团队成员已经基于被删除的标签做了开发，他们可能需要进行调整。因此，最好在团队中通知所有人你打算删除一个标签。
- 在删除远程标签之前，确保你有足够的权限对远程仓库进行更改。
